# 本番環境動作確認チェックリスト

本番環境（Production）へのデプロイ時および運用開始前に確認すべき、環境依存項目のチェックリストです。

## 1. AI 関連 (Vertex AI / Gemini)
- [ ] **Google Cloud Project ID**: 環境変数 `GOOGLE_CLOUD_PROJECT_ID` が正しく設定されているか。
- [ ] **認証キー**: `GOOGLE_APPLICATION_CREDENTIALS_JSON` に有効なサービスアカウントキーが設定されているか。
- [ ] **API有効化**: Google Cloud コンソールで `Vertex AI API` が有効になっているか。
- [ ] **権限**: サービスアカウントに `Vertex AI ユーザー` 権限が付与されているか。
- [ ] **クォータ**: 本番モデル (`gemini-1.5-flash-002`) の1分あたりのリクエスト数制限が運用に耐えうるか。

## 2. データベース・ストレージ (Supabase)
- [ ] **URL/キー**: `SUPABASE_URL`, `SUPABASE_SERVICE_ROLE_KEY`, `SUPABASE_ANON_KEY` が本番環境のものか。
- [ ] **RPC関数**: ベクトル検索用の `match_quotation_files` 関数などが本番DBにデプロイされているか。
- [ ] **Storage Bucket**: `quotation-files` バケツが存在し、適切な RLS ポリシーが設定されているか。
- [ ] **Database Schema**: 開発環境で追加したカラムやテーブル（`credits`, `forum` 等）が本番DBと同期されているか。

## 3. 決済・クレジット (Stripe)
- [ ] **APIキー**: `STRIPE_SECRET_KEY` が `sk_live_...` になっているか。
- [ ] **Webhook**: Stripe Dashboard で本番用 Webhook エンドポイント (`/api/webhook`) が登録され、`STRIPE_WEBHOOK_SECRET` が一致しているか。
- [ ] **価格ID**: `STRIPE_PRICE_LITE` 等の環境変数が、Stripe 本番環境で作成した商品の価格IDと一致しているか。
- [ ] **動作試験**: テストカードではなく、少額のリアル決済でクレジットが自動付与されるか。

## 4. アプリケーション設定
- [ ] **CORS/URL**: `APP_URL` (Frontend) と `API_URL` (Backend) が本番ドメイン (`https://...`) になっているか。
- [ ] **メンテナンス**: `MAINTENANCE_BYPASS_TOKEN` を本番用の推測困難な文字列に変更しているか。
- [ ] **NODE_ENV**: `NODE_ENV=production` となっているか。
- [ ] **PDF Worker**: `unpkg.com` などの外部CDNから PDF.js のワーカーやフォントが正常に取得できるか（企業内ネットワーク制限の確認）。

## 5. 運用監視
- [ ] **ログ**: Railway や Supabase のログコンソールで、致命的なエラー (`Connection Error` 等) が発生していないか。
- [ ] **アクセス制限**: 許可されていないドメインからのAPIアクセスが遮断されているか（CORSの再確認）。
