# AI 図面検索：改善状況と残課題

> 更新日: 2026-03-12

---

## 概要

AI 類似図面検索機能の信頼性と視認性を大幅に向上。
同一図面の即時検出（pHash）と、Gemini API によるリランキングを組み合わせた「ハイブリッド方式」を確立。

---

## 現在のアーキテクチャ

```
[ブラウザ]                          [サーバー]
 ├─ PDF.js で PDF を Canvas に描画
 ├─ 矩形選択 → queryImage (PNG)        ┌──────────────────────────────────┐
 ├─ ターゲットの fileId                │                                  │
 └──────── POST /api/search/similar ──►│ 1. pHash 高速判定 (< 1秒)        │
                                       │    - 距離 0〜2: 即答 (Geminiなし)  │
                                       │    - 距離 3〜15: 候補抽出          │
                                       │ 2. ベクトル検索 (~2秒)            │
                                       │ 3. 名寄せ → 上位10〜20件抽出      │
                                       │ 4. 画像DL & 広範囲クロップ (5倍)   │
                                       │ 5. Gemini ハイブリッドリランク     │
                                       └──────────────────────────────────┘
```

---

## 完了した改善

### 1. pHash ハイブリッド検索の確立
- **高速パス (Fast Path)**: ハミング距離 2 以内の「ほぼ完全一致」は Gemini を呼ばずに即回答（1〜2秒）。
- **精密パス (Hybrid Path)**: 距離 3〜15 の候補は、Gemini が画像として精査。誤判定を劇的に削減。
- **DB連携**: 既存ファイルからの検索時は、クライアントが `fileId` を送信し、サーバーでDB上の有効なハッシュを直接引くことで再解析を回避。

### 2. 検索結果サムネイル（プレビュー）の最適化
- **5.0倍ズームアウト**: 抽出範囲の周囲を広く見せることで、図面のどの部分かひと目で分かるようにしました。
- **中心合わせの補正**: スケールに関わらず、ユーザーが指定した矩形が常にプレビューの中央に来るよう座標系を修正。
- **サーバー側画像処理**: `sharp` による切り出しを正規化座標に対応させ、解像度依存のバグを解消。

### 3. フロントエンドの安定化とノイズ抑制
- **ノイズ遮断**: PDF.js 内部の `XFA` 警告や `Worker terminated` エラーをコンソールから隠蔽。
- **プレビュー不可の解消**: キャンセル時に「プレビュー不可」という誤表示が出ないよう、例外ハンドリングを調整。
- **フォントデータの読み込み**: 標準フォントデータのURLを明示し、描画の欠落を防止。

## 現在の検索性能

| シナリオ | 所要時間 | 精度 | 備考 |
|---|---|---|---|
| 同一図面（pHashヒット） | **~2秒** | 100% | Gemini消費なし |
| 類似図面（ハイブリッド） | ~40〜50秒 | 非常に良好 | Geminiによる目視確認あり |
| 新規ファイル（PDF送信） | ~5秒 | 100% | 登録時と同一エンジンで解析 |
