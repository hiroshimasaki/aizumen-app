# UI/UX 最適化提案書

これまでの開発状況と現在の仕様を踏まえ、ユーザー体験（UX）および業務効率をさらに向上させるための最適化案をまとめました。

## 1. 提案の背景
案件管理システムの主要機能（登録、検索、工程管理、分析）が安定稼働し始めた段階において、データの「見やすさ」から一歩進んだ「使い勝手の良さ」と「業務の気づき（インサイト）」を強化することを目的とします。

## 2. 具体的な提案事項

### ① 検索・一覧画面の質感向上
- **スケルトン・ローディング**: データ読み込み中にカード型のプレースホルダーを表示し、体感速度を改善。
- **検索体験の強化**: ヒット件数ゼロ時の代替提案（「条件をクリア」ボタンなど）の配置。

### ② 工程管理（ガントチャート）の利便性向上
- **ポップオーバー表示**: バーにホバーした際、詳細画面を開かずに納期・数量・顧客名を即座に確認。
- **「本日」表示の強調**: 現在時刻を示すガイド線の視認性を高め、スケジュールの現在地を明確化。

### ③ データ分析のインサイト強化
- **AI要約 / インサイトメッセージ**: 「今月は粗利の高い案件がA社に集中しています」等の判断を助けるコメント表示。
- **グラフからのドリルダウン**: グラフの棒をクリックすると、対象月の案件一覧へフィルタされた状態で遷移。

### ④ 編集フォームの操作性改善
- **スティッキー・アクションバー**: 「保存」「キャンセル」「PDF出力」ボタンを画面下部に固定。長いフォームでもスクロールなしで即座に保存可能に。

### ⑤ ダッシュボードの気づき（プッシュ型通知）
- **アラートバッジ**: 納期直近かつ未着手の案件がある場合、メニューバーにバッジを表示して漏れを防止。

---

## 3. 実装の優先順位（推奨）
1. **④ スティッキー・アクションバー**（日々の入力負荷を即座に軽減）
2. **② ガントチャートのポップオーバー**（工程確認のスピードアップ）
3. **① 検索・一覧の質感向上**（システム全体の洗練度向上）
