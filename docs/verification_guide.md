# 検証手順書：図面類似箇所検索・AI見積もり機能

以下の手順で、実装した新機能の動作確認を行ってください。

## 1. 環境の準備
まず、新しい依存関係とデータベース設定を反映させます。

### 1-1. ブランチの切り替え
`feature/ai-estimation` ブランチにいることを確認してください。

### 1-2. 依存関係のインストール
サーバー側で新しいライブラリ（`sharp`, `@tensorflow/tfjs-node` 代替, `pdf-img-convert` 等）をインストールします。
```bash
# server ディレクトリにて
npm install
```

### 1-3. データベースマイグレーション
`supabase/migrations/00014_enable_pgvector_and_create_drawing_embeddings.sql` を実行して、`pgvector` 拡張の有効化とテーブル作成を行ってください。
※ローカル環境であれば `npx supabase db push`、手動であれば SQL Editor で内容を貼り付けて実行します。

## 2. 動作確認手順

### ステップ1：図面の登録（解析）
1. システムにログインし、新規案件作成（または既存案件編集）画面を開きます。
2. **PDF形式の図面**をアップロードします。
3. サーバーログに `[DrawingSearch] Background indexing complete...` と表示されれば、自動的にタイル分割とベクトル化が行われ、DBに保存されています。

### ステップ2：類似箇所検索
1. アップロードされた図面タイルの上にある**「虫眼鏡（Search）」アイコン**をクリックします。
2. `DrawingSearchModal` が開くので、図面上の検索したい範囲（特定の記号や部品など）を**マウスでドラッグして囲みます**。
3. 「この範囲で検索」ボタンをクリックします。

### ステップ3：Gemini リランキングと AI 見積もりの確認
1. 検索実行後、右側のパネルに候補が表示されます。
2. Gemini が判定した「一致率（％）」と「判定理由（日本語）」が表示されていることを確認します。
3. 候補の「この情報を流用」ボタンをクリックします。

### ステップ4：単価情報の自動反映
1. 選択した過去案件の単価情報（加工費、材料費など）が、現在の案件明細に自動入力されることを確認します。

## 3. トラブルシューティング
- **解析が走らない**: `sharp` や `tensorflow` のインストールが正しく完了しているか確認してください。
- **検索結果が出ない**: 最初にアップロードした図面が正しく DB（`drawing_tiles` テーブル）に格納されているか確認してください。
- **AI判定理由が「判定不能」**: Gemini API キーが正しく設定され、クレジットが残っているか確認してください。

---
> [!TIP]
> 最初の解析には数秒〜数十秒（図面のサイズによります）かかる場合があります。バックグラウンド処理のため、解析中に他の操作を継続することが可能です。
