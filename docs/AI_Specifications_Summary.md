# AI機能 仕様要約ドキュメント (AI Specifications Summary)

本ドキュメントは、AiZumenにおけるAI機能の最新仕様をまとめたものです。システムの改修やモデルの更新時、意図しない精度の低下（デグレード）を防ぐための基準として参照してください。

---

## 1. AI図面類似検索 (Similar Drawing Search)

図面の一部を選択し、過去の図面データベースから形状が一致する箇所を特定する機能です。

### 1-1. 技術スタック & アーキテクチャ
- **処理基盤**: Node.js + Vertex AI / Gemini API
- **アルゴリズム**: 3層ハイブリッド検索方式
    1. **pHash層 (同一性判定)**: 画像全体のdHash値を比較し、同一図面をミリ秒単位で特定。
    2. **ベクトル検索層 (粗絞り込み)**: 各図面のタイル（300x300相当）から抽出したベクトルを `pgvector` で検索。
    3. **AIリランキング層 (精密判定)**: 上位候補を Gemini 1.5 Flash で視覚的に比較。

### 1-2. 重要パラメータ
| パラメータ名 | 設定値 | 役割・影響 |
| :--- | :--- | :--- |
| **paddingFactor** | **8.0** | **[最重要]** AIに渡す候補画像の視野の広さ。小さいと穴の数などの全体像を見落とす。 |
| **pHashThreshold** | 20 (標準) / 2 (高速) | ハミング距離。2以下はAI判定をスキップして同一判定。 |
| **Vector Match Count** | 50 (初期) -> 10 (最終) | ベクトル検索で抽出するタイルの数。 |
| **Tile Size** | 600px (Stride 300px) | 図面をインデックス化する際の分割サイズ。 |

### 1-3. AI判定ロジック (Prompts)
- **判定方針**: 「部分（クエリ）対 全体（候補）」の幾何学的比較。
- **優先順位**: 
    1. 線の接続・孔の数・配置・シルエットの一致（スケール不問）。
    2. 寸法値（φ, M8等）の一致。
    3. 回転（90, 180, 270度）および反転の許容。
- **デグレード時のチェック項目**: AI解析スコアが不当に低い場合、`paddingFactor` が縮小されていないか、プロンプトから「幾何学的構造の重視」の指示が消えていないかを確認してください。

## 2. AI注文書解析 (Order OCR / Document Analysis)

注文書や図面から主要項目を抽出し、データ化する機能です。

### 2-1. 技術スタック
- **モデル**: Gemini 1.5 Flash (Vertex AI 優先)
- **入力形式**: PDF (1ページ目を画像化) または 直接画像

### 2-2. 抽出対象フィールド (JSON Schema)
以下の構造で厳密に抽出するように定義されています。

- **drawingNumber**: 図面番号
- **customerName**: 発注元（自社名は除外）
- **items (明細リスト)**:
    - `name`: 品名
    - `material`: 材質 (SUS304, SS400等)
    - `processingMethod`: 加工方法
    - `surface_treatment`: 表面処理
    - `quantity / unit`: 数量と単位
    - `processingCost`: 加工単価
    - `materialCost`: 材料単価
    - `otherCost`: その他費用
    - `dueDate`: 納期
    - `dimensions`: 寸法 (L x W x T 等)

### 2-3. 指示の重要ポイント (Prompts)
- **検算ルールの導入**: 「数量 × 単価 = 合計」の一致を確認し、不一致がある場合は `requiresVerification` フラグを立てる。
- **正規化**: 数値から通貨記号やカンマを除去し、純粋な数値として出力。
- **材質等の補完**: 図面内の表題欄から正確に材質や熱処理情報を拾う設計。

### 2-4. デグレード時のチェック項目
- 項目が `null` になる、または全く抽出されない場合、プロンプト内の JSON サンプル（`Output Format`）が簡略化されていないか、タイポ（韓国語の混入など）がないかを確認してください。

---

## 3. 運用・保守
- **ログ**: `logs/search_debug.txt` に Gemini の生の応答内容が保存されます。精度に不審な点がある場合はここから AI の推論理由（reason）を確認できます。
- **モデル変更**: プロンプトは Gemini 1.5 Flash 用に最適化されています。上位モデル（Pro等）に変更する場合、トークン消費と精度のバランスを再検証する必要があります。
