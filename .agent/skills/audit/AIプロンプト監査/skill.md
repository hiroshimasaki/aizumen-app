# AIプロンプト監査スキル

あなたは **AI/LLMプロンプトエンジニアリングの専門家** です。
プロンプトの品質、安全性、効率性を体系的に評価し、改善点を洗い出します。

---

## Phase 1: プロンプト構造確認

### 実行手順
1. `grep_search` で `const prompt` や `generateContent` を検索し、全プロンプトを特定
2. 各プロンプトの構造を以下の観点で評価

### チェックリスト
- [ ] **役割定義**: プロンプト冒頭にAI의役割が明確に定義されているか
- [ ] **入力明示**: AIに渡す入力データ（画像、テキスト、数値）が明確か
- [ ] **出力形式指定**: JSON、マークダウンなど出力形式が厳密に指定されているか
- [ ] **出力例**: 具体的な出力例（few-shot）が含まれているか
- [ ] **制約条件**: 禁止事項や制約が明記されているか
- [ ] **言語指定**: 出力言語が指定されているか

---

## Phase 2: ハルシネーションリスク確認

### チェックリスト
- [ ] **根拠の強制**: 「書類に記載されている内容のみ」等の制約があるか
- [ ] **推測の明示**: AIが推測した場合にフラグ（`requiresVerification`等）を立てる仕組みがあるか
- [ ] **不明時の処理**: 情報が見つからない場合のデフォルト値・挙動が定義されているか
- [ ] **数値の検算**: 抽出した数値の整合性チェック（検算ルール）が含まれているか
- [ ] **疑義の報告**: 解析に自信がない場合の報告先フィールド（`systemNotes`等）が指定されているか

---

## Phase 3: 出力安定性確認

### チェックリスト
- [ ] **responseMimeType**: `application/json` 等で形式が固定されているか
- [ ] **temperature**: 創造性が不要なタスクで 0.0〜0.3 に設定されているか
- [ ] **JSONスキーマ**: 出力JSONのキー名が固定されているか（AIに自由にキーを作らせていないか）
- [ ] **型の指定**: 各フィールドの型（string, number, boolean）が明示されているか
- [ ] **日付フォーマット**: 日付形式（YYYY-MM-DD等）が指定されているか
- [ ] **parseJsonResponse**: AIの応答パース時のエラーハンドリングは適切か

### 確認コマンド
```
grep_search:  "temperature" / "responseMimeType" / "parseJsonResponse"
```

---

## Phase 4: 動的値（マッピング・テナント設定）の整合性

### チェックリスト
- [ ] **マッピング参照**: テンプレートリテラル内で `${mappingData.xxx}` として使用されているキーが、`mappingData` オブジェクトに全て定義されているか
- [ ] **キー名一致**: `mappingData` のキー参照が、UIの保存キー名と一致しているか（例: `map.orderNumberLabel` vs `map.orderNumber`）
- [ ] **フォールバック値**: 各マッピング値に適切なデフォルト値（`||` 演算子）があるか
- [ ] **全項目反映**: UIで設定可能な全マッピング項目がプロンプトの該当箇所に挿入されているか
- [ ] **テナント設定の伝達**: 呼び出し元（ルート）からサービスへ必要なテナント設定が全て渡されているか

---

## Phase 5: セキュリティ確認

### チェックリスト
- [ ] **プロンプトインジェクション対策**: ユーザー入力がプロンプトに直接埋め込まれていないか
- [ ] **機密情報の漏洩**: APIキー、内部パス等がプロンプトに含まれていないか
- [ ] **過剰権限**: AIに「何でも回答」系の許可を与えていないか

---

## Phase 6: トークン効率確認

### チェックリスト
- [ ] **冗長な指示**: 同じ意味の指示が繰り返されていないか
- [ ] **不要な例示**: 過剰な few-shot 例がトークンを浪費していないか
- [ ] **画像の前処理**: 画像サイズが最適化されているか
- [ ] **モデル選択**: タスクの複雑さに対して適切なモデルが選択されているか
