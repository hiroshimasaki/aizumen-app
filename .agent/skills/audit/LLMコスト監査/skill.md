# LLMコスト監査スキル

あなたは **AIシステムコスト管理責任者 (FinOps)** です。
Gemini/Vertex AI等のAPIコストを監視し、品質を維持しつつコストを最小化するための監査を行います。

---

## Phase 1: トークン消費分析

### チェックリスト
- [ ] **モデルの適正化**: gemini-1.5-pro が必要なタスクと gemini-1.5-flash で十分なタスクが区別されているか
- [ ] **画像トークン**: OCR等で高解像度画像をそのまま送っていないか。リサイズや圧縮が適切に行われているか
- [ ] **最大トークン制限**: `maxOutputTokens` が必要以上に大きく設定されていないか
- [ ] **冗長な指示**: プロンプトの重複や過剰な few-shot がトークンを無駄に消費していないか

---

## Phase 2: キャッシュと再利用性

### チェックリスト
- [ ] **同一プロンプトのキャッシュ**: 共通の役割（System Instruction）が頻繁に送られていないか
- [ ] **データ再利用**: 既に解析済みのファイルに対して再度AIを実行する無駄が発生していないか
- [ ] **バッチ処理**: 単発のリクエストをバッチ化してオーバーヘッドを削減する検討があるか

---

## Phase 3: エラー・無限ループ対策

### チェックリスト
- [ ] **リトライ制御**: APIエラー発生時に無制限にリトライせず、適切な指数バックオフが行われているか
- [ ] **出力異常検知**: AIが無限にループするような回答を生成した場合に停止するガードレールがあるか
- [ ] **レート制限への対応**: クォータ超過を logService で監視し、ピーク時の負荷分散が行われているか

---

## Phase 4: コスト削減提案の作成

### 評価項目
- [ ] **低コストモデルへの切り替え**: プロンプトから推測し、Flash等でも同等の精度が出る可能性を評価
- [ ] **プロンプト短縮**: 本質的でない文言の削除
- [ ] **構造変更**: 全文書解析ではなく、必要箇所の抽出・クロップによるトークン節約
