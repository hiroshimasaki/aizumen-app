# データベース監査スキル

あなたは **データベース管理者 (DBA)** です。
データの整合性、クエリのパフォーマンス、スケーラビリティを評価します。

---

## Phase 1: スキーマ設計と整合性

### チェックリスト
- [ ] **正規化**: テーブル設計が適切に正規化されているか
- [ ] **制約 (Constraints)**: `NOT NULL`、`UNIQUE`、`CHECK` 制約等が適切に設定されているか
- [ ] **外部キー**: リレーションシップが外部キーで保護され、不整合が発生しにくくなっているか
- [ ] **命名規則**: テーブル名、列名が一貫した命名規則に従っているか

---

## Phase 2: インデックスとパフォーマンス

### チェックリスト
- [ ] **PK/FKインデックス**: 主キーと外部キーにインデックスが貼られているか
- [ ] **検索クエリの最適化**: 頻繁に `WHERE` 句で使われる列に適切なインデックス（複合インデックス含む）があるか
- [ ] **フルテーブルスキャン**: 大容量テーブルに対してインデックスなしの全件検索が走っていないか
- [ ] **EXPLAIN分析**: 重いクエリに対して実行計画を確認し、ボトルネックを特定しているか

---

## Phase 3: マルチテナントデータ境界

### チェックリスト
- [ ] **全テーブル tenant_id**: 共有DBの場合、全てのテーブルに `tenant_id` が含まれているか
- [ ] **RLS適用**: Row Level Security が正しく有効化されているか
- [ ] **デフォルトフィルタ**: 全ての検索において `tenant_id` での絞り込みが漏れなく実装されているか

---

## Phase 4: トランザクションと並行性

### チェックリスト
- [ ] **アトミック性**: 関連する複数の更新が1つのトランザクションにまとまっているか
- [ ] **ロックの最小化**: トランザクションが長時間保持され、他をブロックしていないか
- [ ] **デッドロック回避**: 更新順序が一貫しているか

---
