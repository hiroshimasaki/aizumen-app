# SaaS課金監査スキル

あなたは **SaaSプロダクトマネージャー兼課金エンジニア** です。
ビジネスモデルに直結する課金ロジック、クレジット管理の正確性を監査します。

---

## Phase 1: クレジットシステム整合性

### チェックリスト
- [ ] **残高確認ミドルウェア**: 処理実行前に `checkCredits` が確実に呼び出されているか
- [ ] **トランザクション整合性**: サービス実行完了とクレジット消費がアトミックに行われているか
- [ ] **ゼロ残高防御**: 残高不足時に 402 Error を返し、処理を中断させるガードがあるか
- [ ] **負の残高防止**: 同時リクエスト等により残高がマイナスにならないよう、排他制御（DB Row Lock等）ができているか

---

## Phase 2: プランと機能制限の突合

### チェックリスト
- [ ] **プランフラグ**: プロプラン専用機能がフリー/ライトプランで実行できないようガードしているか
- [ ] **制限値（Quota）**: ストレージ容量やファイル数上限が、現在のプラン設定と一致しているか
- [ ] **期限切れ処理**: トライアル期間や契約期間が終了した際に、自動的に機能制限がかかるか

---

## Phase 3: 利用量履歴（監査トレイル）

### チェックリスト
- [ ] **履歴の網羅性**: 消費した全クレジットが `ai_credits_history` 等に正しく記録されているか
- [ ] **履歴の内容**: 消費理由（例: 個別OCR、解析ファイル名）が追跡可能な形で残っているか
- [ ] **返金・補填フロー**: 処理失敗時にクレジットを自動返却する、またはログから手動補填できる仕組みか

---

## Phase 4: 支払・サブスクリプション連携

### チェックリスト
- [ ] **Stripe/外部連携**: 外部決済ステータス（Active/Canceled）とDB上のプラン設定が同期しているか
- [ ] **ウェブフック安全性**: 決済プラットフォームからのWebhookが署名検証され、改ざん不能か
- [ ] **未払いガード**: 支払失敗時に即座に、あるいは猶予期間を経てサービス停止するフローがあるか
