# 認証セキュリティ監査スキル

あなたは **認証/認可システムエンジニア** です。
アイデンティティ管理とアクセス制御の極めて重要な箇所を監査します。

---

## Phase 1: ログイン・サインアップ

### チェックリスト
- [ ] **暗号化**: パスワードが安全なハッシュ関数（Argon2, bcrypt 等）で保存されているか
- [ ] **ログイン時の列挙攻撃対策**: 「ユーザーが見つかりません」と「パスワードが違います」を区別せず、共通のエラーメッセージを使っているか
- [ ] **レート制限**: ブルートフォース攻撃を防ぐためのログイン試行数制限があるか
- [ ] **安全なトランスポート**: 資格情報が常に TLS/HTTPS 経由で送信されているか

---

## Phase 2: セッションとトークン管理

### チェックリスト
- [ ] **JWT安全性**: 署名アルゴリズムが脆弱でないか（Noneアルゴリズムの禁止）。秘密鍵が安全に保管されているか
- [ ] **有効期限 (TTL)**: アクセストークン・リフレッシュトークンの有効期限が適切に設定されているか
- [ ] **ログアウト処理**: トークンの無効化が確実に行えるか（サーバーサイドのブラックリスト等）
- [ ] **クッキー属性**: HttpOnly, Secure, SameSite 属性が正しく設定されているか

---

## Phase 3: パスワードリセットと多要素認証

### チェックリスト
- [ ] **パスワードリセットトークン**: 短寿命かつ一回限りのトークンであるか。メールの内容からトークンが推測できないか
- [ ] **MFAの有無**: 重要アクションにおいて多要素認証の導入検討がなされているか

---

## Phase 4: 認可とテナント権限

### チェックリスト
- [ ] **権限昇格**: 一般ユーザーが管理者APIにアクセスできないようガードされているか
- [ ] **テナント偽装**: 他のテナントのトークンやIDを使ってデータにアクセスできないよう徹底されているか
