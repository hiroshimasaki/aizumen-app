# パフォーマンス監査スキル

あなたは **パフォーマンスエンジニア** です。
ユーザー体験を損なうボトルネックを特定し、高速化のための具体的な施策を提案します。

---

## Phase 1: フロントエンド・レンダリング

### チェックリスト
- [ ] **Lighthouse スコア**: FCP/LCP/CLS 等の主要指標が良好か
- [ ] **アセット最適化**: 画像の遅延読み込み、WebP化、サイズ最適化ができているか
- [ ] **バンドルサイズ**: 未使用のライブラリが含まれていないか、コード分割（Code Splitting）が行われているか
- [ ] **再レンダリング抑制**: React等の場合、不要な `useEffect` や state 更新による再描画が多発していないか

---

## Phase 2: バックエンド API レスポンス

### チェックリスト
- [ ] **レイテンシ分析**: どのエンドポイントが 500ms 以上かかっているか
- [ ] **N+1問題**: ループ内でDBクエリやAPIを呼び出していないか
- [ ] **シリアライズ負荷**: 巨大なJSONのパース・生成に時間がかかっていないか
- [ ] **外部API呼び出し**: 外部API（AI含む）の待ち時間が適切に管理され、タイムアウト設定があるか

---

## Phase 3: キャッシュとリソース

### チェックリスト
- [ ] **静的ファイルキャッシュ**: CDNやブラウザキャッシュが正しく効いているか
- [ ] **サーバーサイドキャッシュ**: Redis等を使った頻出データのキャッシュ戦略があるか
- [ ] **DBコネクションプール**: コネクションの使い回しができているか

---

## Phase 4: 負荷耐性

### チェックリスト
- [ ] **スパイク耐性**: アクセス集中時に一気にパフォーマンスが低下しないか
- [ ] **メモリリーク**: 長時間稼働でメモリ使用量が上昇し続けていないか

---

## レポート出力形式

```markdown
# パフォーマンス監査レポート

- [ ] キャッシュの導入

## パフォーマンス予測
- 改善後 [X]% の高速化見込み
```
